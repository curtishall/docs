Direct messages with dispatch and other units, plus agency-wide dispatch alerts.

<Note>Screenshot placeholder: Unit chat split view with conversation list and message thread</Note>

## Open

Sidebar → **Chat** (or **Unit Chat**).

## Views

| View | Purpose |
|------|---------|
| **Messages** | Two-way conversations |
| **Dispatch Alerts** | Broadcast notices from dispatch (may show badge count) |

Toggle between **MESSAGES** and **DISPATCH ALERTS** / **ALERTS** in the header.

<Note>Screenshot placeholder: MESSAGES vs DISPATCH ALERTS toggle with alert badge</Note>

## Conversations

### Start a chat

- **Start a conversation** (empty state or toolbar)
- **Start New Chat** — pick a user from the directory

Conversations show:

- Display name (unit or user)
- Last message preview
- Unread badge when new messages arrive

Select a conversation to open the message thread.

<Note>Screenshot placeholder: Conversation list with unread badges</Note>

### Send messages

- Type in **Message {name}…** field and send
- Quick reply chips: **10-4**, **10-8**, **Enroute**, **On Scene**, **Show Me Busy**
- **Attach File** (when enabled in your deployment)

Messages from you appear on the right; incoming messages on the left.

<Note>Screenshot placeholder: Message thread with quick reply chips</Note>

### Search

Filter conversations and message content with the search box when shown in your layout.

## Dispatch alerts

**Dispatch Alerts** lists agency notices:

- **PINNED ALERTS** — stay at top until unpinned or dismissed
- **ALL ALERTS** — remaining active alerts

Each alert card shows category, content, and timestamp. Actions may include **Pin** / **Unpin** and **Dismiss**.

<Note>Screenshot placeholder: PINNED ALERTS and ALL ALERTS sections</Note>

Alerts are separate from call-specific dispatch notes — see [Dispatch notes](/mobile/dispatch-notes).

## Notifications

Unread messages and active alerts contribute to chat notification indicators in the app shell. Opening a conversation marks messages read and clears related notifications for that thread.

## Requires network

Chat and alerts require connectivity. When offline, messaging is unavailable — see [Connectivity and offline](/mobile/connectivity-offline).

## Dispatch CAD counterpart

Dispatchers use **Chat** in **Dispatch CAD** for the same threads, plus **Send SMS** from call detail for text to the public.

See **Dispatch CAD** documentation: [Chat, SMS, and comms](/dispatch/chat-sms-comms).

## Tips

- Use quick replies for common status updates during traffic stops.
- Check **Dispatch Alerts** at shift start for pass-on information.
- Pin critical alerts you need visible all shift.

## Warnings

<Warning>Chat is not a substitute for radio or priority dispatch instructions on active emergencies.</Warning>

<Warning>Do not send CJIS or NCIC query results in chat — use secure agency channels per policy.</Warning>
