## Inline note

1. Open assigned call.
2. **History & Logs** section.
3. **Add Note** in inline field.
4. Success: **Note added successfully**

Empty state: **No history entries yet.**

<Note>Screenshot placeholder: Inline Add Note field</Note>

## Add Dispatch Note dialog

- **Add Dispatch Note** from action menu.
- Hint: **Enter note text...**
- **ADD NOTE**
- Success: **Note saved to call log**

<Note>Screenshot placeholder: Add Dispatch Note dialog</Note>

## Offline behavior

- **No network connection. Note was not sent.** — note is not saved to server.
- Retry when **WS LIVE** or connectivity returns.

See [Connectivity and offline](/mobile/connectivity-offline).

## Tips

- Include unit identifier in note if shared MDT.
- Timestamp is automatic — focus on factual updates.
- Use notes for arrival, suspect description, clearance reasons.

## Restricted / Confidential CJIS notes

If you paste external query response text that starts with a system header such as `FROM-MULES`, `FROM-DOC`, `FROM-NCIC`, `FROM-REVENUE`, or `FROM-NLETS`, Cadmus automatically saves the note as **Restricted / Confidential — Law Enforcement only**.

- **Law Enforcement** agencies see the full note text.
- **Non–Law Enforcement** agencies (Fire, EMS, and similar) see a placeholder: *Restricted / Confidential information - Law Enforcement only* instead of the CJIS content.
- While typing or pasting, the app may show an orange warning before you submit.

This applies to notes added from **History & Logs**, the **Add Dispatch Note** dialog, and Android Auto’s latest-note line (restricted text is not spoken verbatim to non-LE viewers).
