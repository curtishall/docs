What the top bar shows about you and your unit.

<Note>Screenshot placeholder: Top bar with unit pill, officer name, and status chip</Note>

## Top bar identity

| Element | Meaning |
|---------|---------|
| Unit pill | Your assigned call sign / unit from login profile |
| Officer name | Logged-in user display name |
| Status chip | **AVAILABLE**, **ENROUTE**, etc. |

## Setting your unit

- Unit/call sign is assigned by agency admin in **Web Admin** (Users / unit assignment).
- There is **no in-app editor** in Settings to change call sign today.
- If status buttons fail, verify profile unit matches an apparatus on the call.

<Note>Screenshot placeholder: Settings screen showing no unit editor (for doc clarity)</Note>

## Unit device (tablet mode)

Some deployments use shared MDT device login:

- **Activation code** and **Device label (optional)** during enrollment.
- Settings → **Device Settings** → **Disconnect This Device**

<Note>Screenshot placeholder: Device activation screen</Note>

## Organization

Top bar org chip: **Sandbox** vs **Live**

- Switching org shows **Organization updated** dialog when applicable.

<Note>Screenshot placeholder: Organization chip and switch dialog</Note>

## Tips

- Contact dispatch if wrong unit shows after login.
- Badge number in evidence metadata comes from profile — keep admin records current.
- Shared tablets: disconnect device at end of shift when policy requires.

## Warnings

<Warning>Operating under wrong unit affects dispatch board and call assignment.</Warning>
