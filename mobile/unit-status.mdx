Officer availability and call-level status on mobile.

## Dashboard status (off call)

**STATUS CONTROL** on dashboard:

| Chip | Typical use |
|------|-------------|
| **AVAILABLE** | Ready for assignment |
| **BUSY** | Busy but on duty |
| **BREAK** | Meal/break |
| **DISPATCHED** | Has assignment |
| **ENROUTE** | En route |
| **ON SCENE** | On scene |
| **AT JAIL** | At jail |
| **AT OFFICE** | At station |
| **OUT OF SERVICE** | Off service |

Line shows **Current: {status}**

In **live mode**, choosing a status sends `PUT /units/{your-unit-id}/status` to Cadmus so dispatch sees the same availability as the CAD board. If your profile unit cannot be matched to an apparatus, the app shows an error and keeps the previous status.

In **training mode**, status changes stay on the device only.

## Call-level status

On an assigned call, use **DISPATCHED**, **EN ROUTE**, **ON SCENE** — these update your unit on that specific CFS.

See [Call details](/mobile/call-details).

## Profile unit

- Unit/call sign displays in top bar from login profile.
- Set by admin — no in-app editor in Settings today.
- See [Profile and unit](/mobile/profile-and-unit).

## Offline

Status changes require network. Offline attempt: **No network connection. Try again when online.**

## Tips

- Set **AVAILABLE** when ready after clearing a call.
- Use **OUT OF SERVICE** for vehicle issues or end of shift per SOP.
- Do not use **ON SCENE** until physically on location.

## Warnings

<Warning>Wrong status misleads dispatch map and timers — update promptly.</Warning>
