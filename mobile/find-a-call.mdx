## Open

Sidebar **Find a Call**

Title: **Find a Call** — subtitle **Past calls — not NCIC**

## Search fields

Use the compact search header at the top of the screen:

| Field | Use |
|-------|-----|
| **Quick search** | Description, notes, narratives |
| **CFS #** | Exact or partial call number |
| **Address** | Location text |
| **Officer / unit** | Name, badge, or call sign |
| **From / Through** | Date range filter |
| **RTF only** | Report Taken (report to follow) calls only |
| **Sort** | Closed date or reported date (newest first) |
| **More filters** | Person/vehicle fields (name, SSN, DOB, plate, VIN, driver license) |

Tap **Search** to run the query. Tap **Clear** to reset all filters.

Results show **N result(s) — page X of Y** under the filters. On wider screens, results appear in an aligned table; on narrow portrait screens, each result appears as a card.

## Result actions

Each result row or card includes:

| Action | Use |
|--------|-----|
| **Notes** | Add or edit CFS notes for that call |
| **Details** | Open the full call detail screen |

## Open result

Tap **Details** (or tap a result card) to open the call. When you open a result from Find a Call, Cadmus keeps your current result set as a review queue.

- Use **Previous** and **Next** in the call detail header to move through the results.
- Press **Enter** or the **right arrow** to open the next CFS.
- Press the **left arrow** to go back to the previous CFS.
- Keyboard shortcuts are ignored while you are typing in an editable field or dialog.

At the end of a page of results, **Next** loads the first CFS on the next page using the same search filters. At the beginning of a page, **Previous** loads the last CFS on the prior page.

## Tips

- Use CFS number when dispatch radioed it.
- Address search helps repeat-location checks.
- This is **not** NCIC — use **NCIC Search** for criminal queries.

## Warnings

<Warning>Historical call data is sensitive — follow records/PII policy on shared devices.</Warning>
