When a RapidSOS alert appears on the map, you can turn it into a real Call for Service without retyping everything. Cadmus opens **Import RapidSOS Alert** — a focused version of the New CFS form with RapidSOS data already filled in. You review the prefill, choose an incident type, and create the call.

<Note>
  This workflow is for **staged map alerts** (polling or EDX). It is different from **Track Phone Number**, which follows a live caller on the map without creating a CFS.
</Note>

## Before you start

- RapidSOS must be configured for your organization in **Web Admin → Integrations → RapidSOS**.
- **RAD-E** (and optionally **LEI**) must be enabled in **Runtime Settings** for your org.
- A RapidSOS pin must be visible on the map with partner data (see [Map and location — RapidSOS alerts](/dispatch/map-and-location#rapidsos-alerts-on-the-map)).

## Open Import RapidSOS Alert

<Steps>
  <Step title="Click the RapidSOS pin on the map">
    The **RapidSOS Emergency Data** panel opens with caller ID, coordinates, partner sections, and action buttons.
  </Step>
  <Step title="Choose Create New CFS">
    Click **Create New CFS**. Cadmus opens **Import RapidSOS Alert** with pre-filled fields from the alert payload.
  </Step>
  <Step title="Review all sections">
    Use the step bar at the top (**Type**, **Location**, **Nature**, **RP**, **Narrative**) to confirm each section. Green checkmarks mean Cadmus already has data for that section.
  </Step>
  <Step title="Select Incident Type and create">
    **Incident Type** is required — type to search and pick the correct type for your agency. Then click **Create Call** (or **Drop Call** if the alert is not valid).
  </Step>
</Steps>

The import form is intentionally slimmer than a manual New CFS: it emphasizes incident type, location, caller/source, supplemental notes, and optional vehicles — not the full multi-party RP workflow.

## Import form — combined walkthrough

The screenshots below show one continuous form (scroll down through sections). Cadmus does not use a separate “page 2” dialog — location, caller, and vehicles appear below incident detail in the same window.

### Incident detail (Section 1)

<Frame caption="Import RapidSOS Alert — incident type, priority, and pre-filled incident details from the Cove alarm demo.">
  <img src="/images/rapidsos/import-incident-detail.png" alt="Import RapidSOS Alert dialog showing incident detail with Cove alarm prefill and P2 priority" />

| Field | Typical RapidSOS prefill |
|---|---|
| **Incident Type \*** | Empty — dispatcher must select (required before **Create Call** enables) |
| **Priority Level** | May default from alert severity (example: **P2 — Urgent**) |
| **Incident Details** | Short summary from partner data (example: *Cove alarm: Personal Emergency (Living Room)*) |
| **Call time** | Set when the import dialog opens; use refresh if needed |

Footer text shows **Location sent** with the GPS coordinates RapidSOS provided (example: `37.369473, -122.006768`).

### Location and caller (Sections 2–3)

<Frame caption="GPS location, caller phone/name, RapidSOS supplemental notes, and Vehicles Involved on the same import form.">
  <img src="/images/rapidsos/import-location-caller.png" alt="Import RapidSOS Alert showing GPS coordinates, caller fields, supplemental notes, and vehicles section" />
</Frame>

| Section | What you see |
|---|---|
| **2 — Incident Location** | Often **GPS** mode with coordinates pre-filled |
| **3 — Caller / Source** | Caller phone and name; **RapidSOS supplemental notes** text area with medical, contacts, and profile lines |
| **Vehicles Involved** | Populated when crash or mobile partner data includes vehicle fields; use **Add Vehicle** if you need to add or correct one |

Badge chips above the supplemental notes (example: **Cove**, **Location**, **Source**, **Alarm**, **Contacts**, **Medical**) mirror the data categories present in the alert — same categories shown on the map tooltip.

### What is hidden in import mode

To reduce clutter, Import RapidSOS Alert **does not** show the full standard New CFS panels for additional reporting parties, full persons list, or the large RapidSOS insight side panel. Core dispatch fields and vehicles remain available.

## Actions at the bottom of the form

| Button | Result |
|---|---|
| **Cancel** | Closes import without creating a CFS |
| **Drop Call** | Closes and drops the staged alert per agency workflow |
| **Create Call** | Creates the CFS with pre-filled data (requires **Incident Type**) |

<Warning>
  GPS coordinates and medical notes may contain PII. Follow your agency CJIS and privacy policy before sharing outside CAD.
</Warning>
