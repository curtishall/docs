Map and location

Map window, layers, and RapidSOS tracking.

> Screenshot placeholder: Map window with calls and units plotted

## #Open map

- Header **Map** (F4).
 
- May open in separate window on desktop.
 

## #Map layers

**Map Layers** panel toggles:

- **Patrol Units**
 
- **Personnel**
 
- **Active Calls**
 
- **RapidSOS Locations**
 
- **RapidSOS Breadcrumbs**
 

> Screenshot placeholder: Map Layers panel with toggles

## #Map controls

- **Zoom In** / **Zoom Out** / **Reset View**
 
- **Always on Top** (desktop)
 
- **Toggle Fullscreen**
 
- **Select Map Style**
 

> Screenshot placeholder: Map control toolbar

## #Location on calls

From call detail **Location details**:

- Edit address and **Validate and Geocode Address**.
 
- **Copy GPS coordinates** for radio or CAD notes.
 

> Screenshot placeholder: Call detail location block with coordinates

## #RapidSOS tracking

When enabled for your org:

1. **RapidSOS Tracking** or **Track Phone Number**.
 
2. Enter caller phone or select from call context.
 
3. **Start Tracking** — shows emergency location on map.
 
4. **Stop Tracking** when no longer needed.
 

> Screenshot placeholder: RapidSOS tracking panel on map

## #RapidSOS lookup (non-production / when enabled)

- Header **RapidSOS Lookup** dialog.
 
- **Lookup** by phone → **RapidSOS Results** → **Show on Map**.
 

> Screenshot placeholder: RapidSOS Lookup dialog with results

Configure in **Web Admin** → Integrations → RapidSOS.

## #Tips

- Toggle only layers you need — dense maps slow older workstations.
 
- Refresh address resolution after RP corrects location verbally.
 
- Confirm map pin matches cross streets before dispatching.
 

## #Warnings

> **Warning:** RapidSOS availability depends on carrier/data — not a substitute for RP location.

> **Warning:** GPS copy is sensitive — follow CJIS/PII policy when sharing coordinates externally.
