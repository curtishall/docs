Create CFS workflow

Use this guide to create a new Call for Service (CFS) quickly and correctly from the MDT.

Extended version with priority levels and Drop Call notes may be added in a future Dispatch CAD page.

## #Open the Create CFS form

1. Go to the main CAD screen (dashboard).
 
2. In the top bar, select **New CFS** (keyboard **F1**).
 
3. The **Create New Call for Service** dialog opens.
 

## #Required fields

You cannot create a call until all required fields are filled:

- **Incident Type \***: Select a valid incident type from the list.
 
- **Location \***: Enter one of these:
 
 - Street address
 
 - Cross street
 
 - Common place
 
- **Incident Details \***: Enter clear details about what happened.
 

### #Location tips

- **Address** is best when you have a full street location.
 
- **Cross street** is best when no exact address is known.
 
- **Common place** is best for known places saved in your system.
 

## #What each status means

A new CFS is created as **Pending**. Status changes as units are assigned and the call progresses.

- **Pending**: Created, waiting for assignment/dispatch actions.
 
- **Dispatched**: Unit(s) assigned or actively being handled.
 
- **Cleared**: Field handling is complete, call is no longer active.
 
- **Closed**: Finalized record; editing and new logs/attachments are restricted.
 
- **Active** (if shown): Legacy/open state some agencies still use; treat it as an active working call.
 

## #How attachments work

Attachments are added **after** the call is created.

1. Create the CFS first.
 
2. Open that call’s detail page.
 
3. In the **History/Logs** section, add images using:
 
 - Gallery
 
 - File upload
 
 - Paste from clipboard (platform-dependent)
 

### #Attachment rules

- Image size limit is enforced (large files may fail).
 
- Supported image formats are standard image types (JPG/PNG/GIF/WebP).
 
- If a call is **Closed**, new image uploads are disabled.
 

## #Common mistakes

- Typing an incident type without selecting it from the list.
 
- Leaving location blank or entering incomplete location text.
 
- Entering short/vague incident details (for example: "traffic stop" only).
 
- Assuming attachments can be added before saving the call.
 
- Trying to add logs or images after the call is closed.
 

## #Tips for touchscreen MDT use

- Tap the incident type field, type 2-4 letters, then tap a result.
 
- Use the location type that matches what you know first (address vs cross street).
 
- Keep incident details short but specific: who, what, where, immediate risks.
 
- If available, verify address validation feedback before creating the call.
 
- Create first, then immediately add supporting images in History/Logs.
 

## #Warnings

> **Warning:** Do not rely on free-typed incident type text alone. You must select a valid type from the dropdown list.

> **Warning:** Closing a call can block further operational updates (including image/log additions). Confirm the call is truly complete before closing.
