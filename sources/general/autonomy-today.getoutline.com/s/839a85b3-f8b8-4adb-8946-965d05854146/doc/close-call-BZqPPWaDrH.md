Close call

Finalize a CFS when field handling is complete.

> Screenshot placeholder: Close Call dialog with narrative fields

## #When to close

- All assigned units cleared or appropriately removed.
 
- Required narratives entered per agency SOP.
 
- Report numbers assigned if **Report to Follow** applies.
 

## #Close workflow

1. Open call detail for the CFS.
 
2. Select **Close Call** (app bar or call summary).
 
3. Complete dialog fields:
 
 - **Public Narrative** — record suitable for general incident history.
 
 - **Private Notes** — restricted notes when supported.
 
 - **Select Primary Officer:** — reporting officer when required.
 
4. Confirm close.
 

> Screenshot placeholder: Primary officer dropdown in close dialog

## #Report to follow

- Closing with RTF may assign report/case numbers per agency settings.
 
- See **Web Admin** → Citation and report settings.
 

## #After close

- Call moves to **Recent** board or drops from active tabs.
 
- New logs and images typically disabled.
 
- Use **Reopen Call** only with supervisor approval.
 

> Screenshot placeholder: Closed call detail showing restricted edit state

## #Merge before close

If duplicate CFS exists, consider **Merge into…** instead of closing both separately.

## #Common mistakes

- Closing with units still assigned.
 
- Empty public narrative when SOP requires minimum text.
 
- Closing in wrong agency primary when multi-agency call.
 

## #Warnings

> **Warning:** Close is difficult to undo — verify call number and location before confirming.

> **Warning:** Closed calls may block operational updates — confirm with field units first.
